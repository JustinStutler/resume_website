## 2. Key architecture decisions (confirmed with user)

- **Vault delivery:** Move the **wiki portion only** into `client/` and serve the raw `.md` files live; the client renders them with marked.js. **No content duplication.**
- **Only the wiki is public.** `sources/` + meta (`VAULT_SCHEMA.md`, `index.md`, `log.md`) stay at the repo root, **never** under `client/` (Netlify serves everything in the publish dir, so they must physically not be there).
- **Generated index:** one small `wiki-index.json` (**metadata only** — id, path, title, summary, tags, parent, related). Bodies fetched live on click. Search matches title/summary/tags. Regenerated by a script; committed so Netlify still needs **no build command**.
- **No wiki backend endpoint** — it's all public info and would incur Render cold starts.
- **Backend keeps working** by reading from two roots (moved wiki dir + sources dir), preserving current AI retrieval behavior exactly.
- Verified: all internal wiki links target wiki pages → wiki-only client index has **zero broken links**.

## 4. Phases

**Phase 0 — Safety branch.** ✅ Done.

**Phase 1 — Vault relocation + index generator (backend).**
- `git mv Portfolio_Website_Vault/wiki client/vault/wiki`.
- Update `vault_loader.py` to walk two roots (moved wiki + `Portfolio_Website_Vault/sources`). Keep `sources/` loaded so AI behavior is unchanged.
- Add `server/gen_wiki_index.py`: reuse the loader's frontmatter parser, emit `client/vault/wiki-index.json` (metadata only, wiki pages only). Include a home flag for `justin-stutler`.
- Update `test_vault_integrity.py` path expectations; run it. Regenerate index. Restart server, ask a sample query end-to-end to confirm AI still answers.

**Phase 2 — Chat formatting fix (backend, independent).**
- Rewrite the `FORMATTING` block of `ANSWER_SYSTEM_INSTRUCTION`: short topic title (a few words) or none, fluid prose, light markdown (bold/lists, tables only for genuinely tabular data), drop "largest most prominent element." Keep third-person + context-only rules.
- Verify with 2-3 sample queries that replies no longer open with a giant heading.

**Phase 3 — Wiki frontend shell (the big one).**
- Rewrite `index.html` into a wiki shell. **Top bar (one level): search bar + 6 section-nav headers** repurposed from the old colored column titles, remapped to wiki sections: Academics→`academics`, Career→`career`, Projects→`projects`, Skills→`skills`, Personal→`personal`, About→`about-this-website`. Each header links to its section hub; keep the color accents. Below the top bar: the article container; docked chat region at the bottom.
- **Remove all suggestion buttons + columns + the GRE `/\bgre\b/` intercept** (from `index.html` and `handlers.js`).
- `router.js`: hash routing `#/wiki/<id>`, default → `justin-stutler`, back/forward support.
- `wikiView.js`: fetch `vault/wiki/<path>.md` (path from index), parse frontmatter, render body via marked.js, build an in-article "Contents" nav, and on the home page render an **infobox with `ice_cave.JPG`** in the top-right.
- `links.js`: post-render, rewrite bare-id anchors to `#/wiki/<id>`; unknown ids render inert.
- `search.js`: load `wiki-index.json`, live-filter titles/summaries/tags, dropdown → navigate.

**Phase 4 — Docked chat integration.**
